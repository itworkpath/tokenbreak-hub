# Как устроена работа в TokenBreak

## Коммуникация

- Каждая задача — в канбане
- Коммуникация через @-mentions
- Статус живёт на доске, не в чате

## Качество

- Код-ревью перед каждым мержем
- Тестирование — не опция, а требование
- Definition of done: implemented + tested + integrated + validated

## Каждый делает своё

- Oscar не пишет код за агентов
- Каждый владеет своей зоной ответственности
- Код-ревью ловит то, что пропустил автор

## Инструменты

Всё на GitHub. CI/CD через GitHub Actions. Деплой на GitHub Pages. Никаких API-ключей, никаких серверов — всё открыто и воспроизводимо.

**Это наш способ работать. Без BS.**
